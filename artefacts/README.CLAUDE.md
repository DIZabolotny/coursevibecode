# README — `CLAUDE.md`

## Что это

`CLAUDE.md` — это основной операционный документ проекта, в котором задана методология автономного извлечения содержимого онлайн-курсов в структурированный Markdown.

Файл описывает не просто формат данных, а полный рабочий протокол для AI-агента:
- как начать обработку курса;
- как извлекать структуру курса;
- как проходить лекции последовательно;
- как получать транскрипты;
- как сохранять скриншоты и ресурсы;
- как собирать индекс курса и итоговое summary;
- как поддерживать репозиторий в консистентном виде.

По сути, это **инструкция для автономного агента**, который должен пройти курс целиком почти без участия человека.

---

## Для чего нужен этот файл

Этот файл нужен в тех случаях, когда вы хотите превратить курс из закрытой платформы обучения в локальную, хорошо организованную базу знаний.

Типовые задачи:
- сохранить купленный курс в виде текстовой базы знаний;
- сделать содержимое курса доступным для поиска;
- подготовить материал к конспектированию, сжатию, анализу и переиспользованию;
- получить лекции в читаемом и единообразном формате;
- собрать summary по курсу без ручной рутины.

## Что именно задаёт `CLAUDE.md`

Файл определяет:

### 1. Цель проекта
Главная цель — получить локальные копии курса в виде:
- транскриптов,
- скриншотов,
- ресурсов,
- индексов,
- итоговых summary.

### 2. Правила автономной работы
Агент должен работать автономно и не дёргать пользователя на каждом шаге. Прерывание допустимо только в критических случаях, например:
- истекла сессия и нужен повторный логин;
- встретился неизвестный тип лекции;
- инструмент сломался и не восстанавливается после нескольких попыток.

### 3. Стандартную структуру папок
Файл задаёт каноническую структуру репозитория:
- отдельная папка на каждый курс;
- отдельные папки на секции;
- один Markdown-файл на каждую лекцию;
- папка `screenshots/`;
- индекс курса;
- summary курса;
- корневой `README.md`.

### 4. Соглашения по именованию
Здесь задаются правила для:
- имени папки курса,
- имени папки секции,
- имени лекции,
- имени скриншота.

Это критично для предсказуемости структуры и дальнейшей автоматизации.

### 5. Шаблон файла лекции
Каждая лекция должна содержать стандартные поля:
- Course
- Section
- Lecture
- Duration
- URL
- Transcript
- Screenshots
- Resources & Links
- Notes

### 6. Пошаговый workflow
Описан полный pipeline:
1. собрать структуру курса;
2. открыть лекцию;
3. активировать transcript;
4. извлечь текст;
5. проверить ресурсы;
6. сделать скриншоты;
7. сохранить файл;
8. перейти к следующей лекции;
9. в конце собрать индекс и summary.

### 7. Обработку edge cases
Файл заранее задаёт правила для нестандартных ситуаций:
- нет транскрипта;
- лекция — quiz;
- transcript не открылся;
- transcript автоскроллится;
- страница требует повторного логина;
- очень длинный transcript.

### 8. Финальные критерии качества
В конце перечислен quality checklist:
- все лекции сохранены;
- индекс собран;
- summary собрано;
- ссылки работают;
- названия единообразны;
- нет чувствительных данных.

## Когда использовать

Используйте `CLAUDE.md`, если вы хотите:

- автоматизировать разбор Udemy-курсов;
- поручить Claude / Cowork / browser-agent работу “под ключ”;
- получить воспроизводимый pipeline;
- стандартизировать формат учебного архива;
- минимизировать ручную работу при извлечении курса.

## Что требуется для работы

Перед использованием обычно нужно:

1. Иметь доступ к курсу.
2. Открыть курс в браузере.
3. Иметь рабочее расширение для браузерной автоматизации.
4. Создать локальную папку проекта.
5. Положить туда этот `CLAUDE.md`.
6. Настроить локальные разрешения в `.claude/`.

## Сильные стороны файла

### Чёткая операционная логика
Файл не абстрактный, а прикладной: он описывает реальные действия шаг за шагом.

### Хорошая масштабируемость
Можно обрабатывать не один курс, а серию курсов, не теряя структуры.

### Подходит для автономного режима
Документ явно написан так, чтобы агент мог работать с минимальным supervision.

### Хорошая база для последующего анализа
Результат легко использовать для:
- поиска,
- конспектирования,
- summarization,
- knowledge extraction,
- RAG-пайплайнов.

## Ограничения

### Привязка к конкретному workflow
Документ сильно завязан на связку браузер + Udemy + transcript panel + Claude/Cowork-инструменты.

### Возможная хрупкость UI-автоматизации
Если Udemy меняет интерфейс, часть шагов может устареть.

### Нет формального machine-readable spec
Сейчас это текстовая инструкция для агента, а не строгая схема в JSON/YAML.

### Есть несогласованность в имени settings-файла
В одном месте фигурирует `.claude/settings.local.json`, а в `artefacts` лежит `setting.json`. Это стоит привести к одному стандарту.
