# README — `learningwithcowork.md`

## Что это

`learningwithcowork.md` — это объясняющий текст и практический кейс о том, как использовать Claude + Cowork для почти полностью автономного извлечения и структурирования учебных курсов.

В отличие от `CLAUDE.md`, который задаёт строгий рабочий протокол для агента, этот файл выполняет другую роль:

- показывает реальный эксперимент;
- объясняет, зачем вообще нужен такой подход;
- демонстрирует масштаб экономии времени;
- вводит пользователя в setup;
- мотивирует повторить workflow у себя.

Иначе говоря:

- `CLAUDE.md` — это **инструкция для агента**;
- `learningwithcowork.md` — это **объяснение для человека**.

---

## Основная идея файла

Файл показывает, что AI можно использовать не только как чат, а как автономного исполнителя длинной рутинной работы.

На примере трёх Udemy-курсов описан сценарий, где агент:

- сам открывает лекции;
- извлекает транскрипты;
- создаёт Markdown-файлы;
- формирует summaries;
- обновляет README;
- адаптируется к проблемам;
- в одном из случаев самостоятельно находит обход через внутреннее API.

Это делает файл сильным демонстрационным материалом для идеи “AI не просто отвечает, а работает”.

---

## Для чего нужен этот файл

Файл нужен для трёх задач.

### 1. Демонстрация возможности
Он показывает, что автономная работа агента уже практична, а не теоретична.

### 2. Онбординг пользователя
Файл помогает человеку понять:
- что надо установить;
- какие файлы подготовить;
- как организовать папку;
- как запустить процесс.

### 3. Маркетингово-образовательная функция
Это не только инструкция, но и сильный narrative-driven кейс, который продаёт сам подход через результат и масштаб выигрыша.

## Что содержится внутри

Файл включает:

### Описание эксперимента
Указывается:
- сколько было курсов;
- сколько лекций;
- сколько часов видео;
- сколько текста извлечено;
- сколько времени заняла автономная работа;
- сколько времени сэкономлено относительно ручного способа.

### Объяснение workflow
Описано, как агент работал через браузерное расширение и DOM-доступ.

### Ключевой момент адаптации
Отдельно подчёркивается важная вещь: когда UI-путь сломался, агент сам нашёл альтернативный путь через внутреннее API и VTT-субтитры.

Это центральная доказательная часть документа: он показывает не только automation, но и problem-solving.

### Setup-инструкции
Файл объясняет, что нужно сделать до запуска:
- оформить подписки;
- установить расширение;
- подключить desktop app;
- создать папки;
- положить `CLAUDE.md`;
- создать `.claude`-настройки.

## Ограничения

### Текст смешивает несколько ролей сразу
Сейчас в одном файле перемешаны:
- кейс-стади,
- инструкция,
- мотивационный текст,
- setup-гайд.

Из-за этого документ чуть менее чистый по структуре, чем мог бы быть.

### Недостаточно формализации
Некоторые шаги описаны разговорно. Для повторяемости хорошо бы добавить:
- checklist;
- prerequisites;
- troubleshooting;
- expected outputs;
- explicit warnings.

### Язык и аудитория
Файл написан в ярком публицистическом стиле, что хорошо для вовлечения, но хуже для сухой технической документации.

## Что можно улучшить в первую очередь

### 1. Разделить “историю” и “инструкцию”
Сначала короткий кейс, потом чёткий setup.

### 2. Добавить предупреждения
Например:
- нужен легальный доступ к курсам;
- интерфейс Udemy может меняться;
- некоторые лекции могут не иметь transcript;
- часть workflow зависит от активной вкладки или browser automation.

### 3. Привести naming в порядок
Нужно синхронизировать имена:
- `setting.json`
- `settings.local.json`

### 4. Добавить финальный checklist
Чтобы пользователь понимал, что должно получиться в конце.
