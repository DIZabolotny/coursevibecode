## Тезаурус: подробное описание каждой возможности

### Базовые возможности

**Текстовый диалог с моделью.** Основной способ взаимодействия с Claude: вы пишете запрос, модель отвечает. Доступен везде. Различия между средами начинаются не в самом диалоге, а в том, что модель может *сделать* помимо ответа текстом.

**Загрузка файлов в диалог.** Возможность прикрепить файл (PDF, изображение, код, документ) к сообщению, чтобы модель его проанализировала. В Chat и Cowork — через интерфейс drag-and-drop. В Claude Code — файлы уже доступны через файловую систему, но можно и указать путь явно.

**Веб-поиск.** Модель может самостоятельно искать информацию в интернете для ответа на вопрос. В Chat доступен на тарифе Pro. В Cowork и Claude Code — встроен как инструмент (WebSearch/WebFetch).

**Артефакты (React, HTML, SVG).** Интерактивные превью кода прямо в интерфейсе: React-компоненты с Tailwind, HTML-страницы, SVG-графика, Mermaid-диаграммы. Артефакт — это не файл на диске, а рендер в окне чата. Удобно для прототипирования интерфейсов, визуализации данных, создания интерактивных демо. В Claude Code этой функции нет, потому что там вы работаете в терминале, а не в графическом интерфейсе.

**Projects (контекст + инструкции).** Функция Chat, позволяющая создать «проект» — папку с набором файлов и системных инструкций, которые автоматически подгружаются в каждый диалог внутри проекта. Полезно, когда вы регулярно работаете над одной темой и не хотите каждый раз объяснять контекст заново. В Claude Code аналогичную роль выполняет файл CLAUDE.md в корне проекта.

### Файловая система

**Linux VM с bash-терминалом.** Cowork и Claude Code имеют доступ к полноценной командной строке. Это значит, что модель может выполнять любые команды: устанавливать программы, запускать скрипты, обрабатывать данные, конвертировать форматы. Принципиальная разница: в Cowork это изолированная виртуальная машина (песочница), в Claude Code — ваш реальный терминал на вашем компьютере.

**Создание файлов (docx, pptx, xlsx, pdf).** Модель может создавать реальные файлы офисных форматов, которые вы скачиваете и открываете в Word, Excel, PowerPoint или любом PDF-ридере. В Chat этого нет — там только текст и артефакты. В Cowork и Claude Code файлы создаются через Python-библиотеки (python-docx, openpyxl, python-pptx и др.).

**Монтирование папки пользователя.** В Cowork вы можете выбрать папку на своём компьютере, и модель получит к ней доступ — сможет читать ваши файлы, редактировать их, создавать новые. В Claude Code аналогичный эффект достигается тем, что вы запускаете его в нужной директории, и он видит всё дерево файлов.

**Чтение/редактирование файлов на диске.** Модель может открыть существующий файл, прочитать его содержимое, внести точечные правки и сохранить. Это принципиально отличается от загрузки файла в диалог: при загрузке модель видит содержимое, но не может изменить оригинал. При работе через файловую систему — может.

**Установка пакетов (pip, npm).** Модель может самостоятельно установить недостающую библиотеку. Например, если для обработки Excel нужен openpyxl, она выполнит `pip install openpyxl`. В Chat такой возможности нет — модель работает только с тем, что уже доступно.

**Запуск произвольных скриптов.** Возможность написать скрипт на Python, JavaScript, Bash или любом другом языке и тут же его выполнить. Это делает модель не просто генератором текста, а исполнителем задач: обработка данных, парсинг, конвертация, автоматизация.

### Интеграции

**MCP-коннекторы (Slack, GitHub и др.).** MCP (Model Context Protocol) — открытый протокол, позволяющий подключать модель к внешним сервисам. Через MCP-коннекторы модель может читать сообщения в Slack, создавать задачи в Jira, работать с GitHub-репозиториями, обращаться к CRM и десяткам других систем. Коннектор — это «мост» между моделью и внешним API. Без коннектора модель не имеет доступа к вашим рабочим инструментам.

**Claude in Chrome (управление браузером).** Функция Cowork, дающая модели возможность управлять браузером Chrome: переходить по ссылкам, читать содержимое страниц, нажимать кнопки, заполнять формы, делать скриншоты, выполнять JavaScript. По сути — браузерная автоматизация, аналог Selenium или Puppeteer, но управляемая через естественный язык. Полезна для задач, которые требуют взаимодействия с веб-интерфейсами, для которых нет API или MCP-коннектора.

**Figma-интеграция.** MCP-коннектор к Figma, позволяющий модели получать скриншоты макетов, метаданные компонентов, дизайн-токены и переменные. Полезен для генерации кода по макетам, анализа дизайн-системы, проверки соответствия реализации макету.

**Git-операции.** Работа с системой контроля версий: просмотр истории коммитов, создание веток, коммиты, push, создание pull request. В Cowork доступны базовые git-команды, но полноценная работа (включая push на удалённый репозиторий, разрешение конфликтов, ребейз) возможна только в Claude Code, где модель работает напрямую с вашим репозиторием.

**API-вызовы из терминала.** Возможность обращаться к любому API через curl, httpie или скрипты. Модель может отправлять HTTP-запросы, получать данные, обрабатывать JSON-ответы. Это универсальный способ интеграции с любым сервисом, у которого есть API, даже если для него нет готового MCP-коннектора.

### Автоматизация

**Scheduled Tasks (планировщик).** Встроенный планировщик задач в приложениях Cowork и Claude Code Desktop. Позволяет создавать задачи, которые выполняются по расписанию: ежечасно, ежедневно, по будням, еженедельно. Каждый запуск — отдельная сессия Claude, которая выполняет заданный промпт. Важное ограничение: планировщик работает только при запущенном приложении. Если приложение закрыто, задачи не выполняются, но пропущенный запуск сработает при следующем открытии.

**Системный cron / launchd.** Планировщик задач на уровне операционной системы. В отличие от Scheduled Tasks, работает независимо от любых приложений — достаточно, чтобы компьютер был включён. Claude Code может создать cron-задачу (Linux) или launchd plist (macOS), которая по расписанию запускает скрипт или команду `claude -p "промпт"`. Это полноценная серверная автоматизация на локальной машине.

**GitHub Actions.** Облачная автоматизация на серверах GitHub. Claude Code может быть интегрирован в CI/CD-пайплайн: ревью кода при каждом pull request, автоматическое исправление ошибок, генерация документации при мерже. Работает полностью независимо от вашего компьютера — задачи выполняются в облаке.

**Headless-режим (без интерфейса).** Возможность запустить Claude Code без интерактивного терминала, передав промпт как аргумент: `claude -p "проанализируй логи за сегодня"`. Модель выполнит задачу и вернёт результат в stdout. Это ключевая функция для встраивания Claude Code в скрипты, пайплайны и любую автоматизацию.

**Agent SDK (программный контроль).** Библиотека (Python / TypeScript), предоставляющая возможности Claude Code в виде программного API. Вы пишете код, который создаёт агента, даёт ему инструменты (чтение файлов, выполнение команд, обращение к API), и агент автономно выполняет задачу. Это конструктор для создания собственных AI-агентов, которые могут работать как сервисы, обрабатывать очереди задач, запускаться в Docker-контейнерах. Максимальный уровень гибкости и контроля.

**Хуки (pre/post tool execution).** Механизм перехвата вызовов инструментов в Claude Code. Позволяет выполнить произвольный код до или после того, как модель вызовет инструмент. Например: логировать все bash-команды, блокировать опасные операции (`rm -rf`), добавлять проверки перед записью файлов, отправлять уведомления при определённых действиях. Инструмент для продвинутых пользователей, которые хотят контролировать поведение агента.

### Skills и Workflow

**Предустановленные Skills.** Набор готовых workflow для типовых задач: создание документов (docx, pptx, xlsx, pdf), финансовый анализ, юридическая проверка, customer support, product management. Каждый skill — это файл с подробными инструкциями, шаблонами и best practices, который подгружается перед выполнением задачи. В Cowork skills встроены и активируются автоматически по ключевым словам в запросе. В Claude Code аналогичный эффект достигается через файл CLAUDE.md, в который можно включить любые инструкции.

**Плагины (устанавливаемые пакеты).** Механизм расширения Cowork: устанавливаемые пакеты, которые добавляют новые skills, MCP-коннекторы и инструменты. Аналог «магазина приложений» для рабочего пространства модели. Плагины могут быть созданы сообществом или организацией и распространяться через реестр.

**CLAUDE.md (настройки проекта).** Текстовый файл в корне проекта, который Claude Code читает при запуске. Содержит инструкции, контекст, правила, описание архитектуры — всё, что модель должна «помнить» при работе с этим проектом. Фактически это персонализированный системный промпт, привязанный к конкретному проекту, а не к сессии.

**Todo-трекинг (виджет).** Визуальный виджет в интерфейсе Cowork, отображающий список задач и прогресс их выполнения. Модель автоматически обновляет его в ходе работы, показывая, какие шаги выполнены и какие остались. Полезно для многошаговых задач, где важно видеть текущее состояние.

**Субагенты (параллельные задачи).** Возможность модели порождать дочерние агенты для выполнения подзадач. Каждый субагент имеет свой контекст, набор инструментов и может работать параллельно с другими. Например: основной агент анализирует задачу, порождает субагента для проверки безопасности кода и субагента для запуска тестов — оба работают одновременно.

**Кастомные slash-команды.** Возможность создавать собственные команды (например, `/deploy`, `/review`, `/report`), которые запускают предопределённый workflow. По сути — ярлыки для часто используемых сценариев.

### Разработка

**Полный доступ к файловой системе ОС.** Claude Code работает непосредственно в вашей операционной системе, без песочницы. Он видит все файлы, к которым у вас есть доступ, может переходить между директориями, работать с несколькими проектами. Cowork, напротив, ограничен виртуальной машиной и видит только подмонтированную папку.

**Запуск тестов и линтеров.** Возможность выполнить `pytest`, `npm test`, `eslint`, `mypy` и любые другие инструменты проверки кода. Claude Code может запустить тесты, прочитать результаты, найти и исправить ошибки, запустить тесты повторно — в полном цикле. В Cowork это возможно ограниченно (внутри песочницы, без доступа к реальному проекту).

**Рефакторинг кодовой базы.** Масштабные изменения в коде: переименование переменных во всём проекте, вынесение функций, изменение архитектуры, миграция между библиотеками. Claude Code видит всю кодовую базу и может вносить согласованные изменения в десятки файлов. В Cowork возможности ограничены объёмом песочницы.

**Git commit / push / PR.** Полный цикл работы с git: создание осмысленных коммитов, push на удалённый репозиторий, создание pull request через GitHub CLI. Claude Code может самостоятельно оформить результаты своей работы как PR с описанием, что позволяет встроить его в стандартный workflow разработки.

**Отладка в реальном проекте.** Возможность запустить приложение, воспроизвести баг, прочитать логи, поставить отладочные выводы, исправить код и проверить результат — всё в реальном окружении проекта, с реальными зависимостями и конфигурацией.

**Docker / контейнеры.** Работа с Docker: сборка образов, запуск контейнеров, просмотр логов, отладка docker-compose конфигураций. Доступно только в Claude Code, так как требует доступа к Docker daemon на хост-машине.

### Целевой пользователь

**Chat** рассчитан на самую широкую аудиторию: любой человек, которому нужно задать вопрос, написать текст, проанализировать документ или получить совет. Не требует никаких технических навыков.

**Cowork** рассчитан на продвинутых пользователей и специалистов, которым нужно создавать файлы, работать с документами, подключаться к рабочим инструментам через коннекторы и автоматизировать повторяющиеся задачи — но без навыков программирования.

**Claude Code** рассчитан на разработчиков и технических специалистов, которые работают в терминале, пишут код, управляют инфраструктурой и хотят максимальный контроль над поведением модели.
