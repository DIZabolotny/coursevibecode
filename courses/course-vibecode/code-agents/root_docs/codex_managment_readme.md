# Codex: краткое введение

> Версия файла: `v1.0`
> Дата версии: `2026-03-16`
> Тип документа: `краткое введение`
> Связанный полный документ: [codex_managment.md](codex_managment.md)
>

Этот документ — короткая входная версия к полному гайду [`education/root_docs/codex_managment.md`](codex_managment.md).

Он объясняет, как смотреть на `Codex` не как на “модель для кода”, а как на агентный runtime, поведение которого определяется сочетанием `config.toml`, `AGENTS.md`, `skills`, `subagents`, `rules`, `sandbox`, `MCP`, `automations`, `exec` и `SDK`.

Ключевая идея состоит в том, что управляемая автономность в `Codex` возникает только тогда, когда инструкции, policy-слой и execution-слой разделены и настроены явно. `AGENTS.md` и skills задают рабочий контекст, `config.toml`, approvals, sandbox и rules ограничивают действия, а `MCP`, `exec`, automations и SDK расширяют среду исполнения.

Полный документ нужен тем, кто хочет:
- использовать `Codex` как инженерную систему, а не как просто coding assistant;
- настраивать воспроизводимую конфигурацию под свои проекты;
- проектировать subagents и reusable skills;
- контролировать shell-политику, внешние инструменты и execution boundaries;
- выстраивать локальные, фоновые и программные агентные workflows.

Если нужен быстрый маршрут чтения, начинайте с разделов `Главные линии`, `Архитектурная схема`, `Схема прохождения задачи`, а затем переходите к разделам про `config.toml`, `AGENTS.md`, `skills`, `subagents`, `rules`, `sandbox`, `MCP` и `exec`.
